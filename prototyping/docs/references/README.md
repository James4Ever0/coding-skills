# Prototyping References

Reference files are organized by **topic/domain**, not by language.

## How to use

1. Before starting a prototype, search the web for existing projects, demos, or reference implementations.
2. Ask the user if they have specific examples they want you to follow or imitate.
3. Save relevant links, patterns, and notes in the appropriate topic file below.
4. When you discover new patterns during implementation, add them here.

## Topic Index

| File | Covers |
|------|--------|
| `cli-parsing.md` | CLI argument parsing patterns across languages |
| `file-io.md` | File reading/writing, path handling, streaming |
| `testing.md` | Testing patterns, assertions, test organization |

## Adding a new topic

Create a new `<topic>.md` file and link it from this README. Keep files focused — split if they grow beyond ~100 lines.

## Maintenance

Update reference files when you:
- Discover a new pattern during prototyping
- Find a useful reference project
- Learn a language-specific idiom worth remembering
